Operators overview

"Operators" tell autosyntax which type of code to generate.
{% br 1 %} {# > O V E R V I E W FOR#}
  • Operators are almost always built-in elements from the python standard library, so that no learning is needed.
  • In a given line, the Operator is the first argument.
  • Currently available Operators:
{% br 7 %} {# D O T T E D A R G U M E N T S#}
.Dotted arguments

Mark an argument with a dot to toggle a less common use-case.

With basic operators, arguments turn to strings.
"Dotting" and argument denotes a variable.
{# {% basic_text_ind 'With basic operators, arguments turn to strings.
"Dotting" and argument denotes a variable.' %}#}
list milk nutella PRESS TO TOGGLE EXAMPLE
[ 'milk' , 'nutella' ]
list milk .nutella PRESS TO TOGGLE EXAMPLE
[ 'milk' , nutella]

These cases are reversed; a dot makes a string.

dict when_im .64 PRESS TO TOGGLE EXAMPLE
{ 'when_im' : '64' }

Inline methods()
Type a built-in method before an argument to apply it.

This is useful in cases like:
for enumerate things
for i , thing in enumerate (things):
{% br 2 %}
or:
my_dict = dict zip keys values
my_dict = dict ( zip ( keys , values)):

Complex operators
{# >D E F#}
def

  • 1st argument is the method's name
  • 2nd+ arguments are the method's parameters
  • Dotting an argument makes it the default value of the previous one
  • A built in method sets the type of the previous argument

fun stuff:
  • 'args' and 'kwargs' are translated to *args and **kwargs
  • If indentation is found, a 'self' is added as first parameter
  • Magic methods are wrapped in __dunder__

{% br 2 %} {# >C L A S S#}
class
  • 1st argument is the class's name
  • 2nd+ arguments are its super classes
  • Typing 'init' creates an __init__ method
  • Following 'init', insert parameters, typings and default values

fun stuff:
  • 'init' arguments are automatically set as data members
  • If any super class was typed, its __super__ is called
  • Type multiple super classes preceding 'init' for mutiple inheritance
